# <a name="replenishment"></a><span data-ttu-id="0c274-103">Aanvulling</span><span class="sxs-lookup"><span data-stu-id="0c274-103">Replenishment</span></span>

[!include[banner](../includes/banner.md)]

<span data-ttu-id="0c274-104">Dit onderwerp beschrijft de aanvullingsstrategieën die beschikbaar zijn voor magazijnen die gebruikmaken van de functionaliteit die beschikbaar is in Magazijnbeheer.</span><span class="sxs-lookup"><span data-stu-id="0c274-104">This topic describes the replenishment strategies that are available for warehouses that use the functionality that is available in Warehouse management.</span></span> <span data-ttu-id="0c274-105">De informatie in dit onderwerp is niet van toepassing op de magazijnoplossing die in Voorraadbeheer beschikbaar is.</span><span class="sxs-lookup"><span data-stu-id="0c274-105">The information in this topic doesn't apply to the warehousing solution that is available in Inventory management.</span></span>

<span data-ttu-id="0c274-106">De volgende aanvullingsstrategieën zijn beschikbaar:</span><span class="sxs-lookup"><span data-stu-id="0c274-106">The following replenishment strategies are available:</span></span>

- <span data-ttu-id="0c274-107">**Aanvulling voor wave-vraag** – Deze strategie maakt aanvullingswerk voor uitgaande orders of ladingen als geen voorraad beschikbaar is wanneer werk wordt gemaakt door de wave.</span><span class="sxs-lookup"><span data-stu-id="0c274-107">**Wave demand replenishment** – This strategy creates replenishment work for outbound orders or loads if inventory isn't available when the wave creates work.</span></span> <span data-ttu-id="0c274-108">Zo kan bijvoorbeeld aanvullingswerk worden gemaakt als de hoeveelheid die is vereist voor een verkooporder niet beschikbaar is wanneer een wave wordt verwerkt.</span><span class="sxs-lookup"><span data-stu-id="0c274-108">For example, replenishment work can be created if the quantity that is required for a sales order isn't available when a wave is processed.</span></span>
- <span data-ttu-id="0c274-109">**Min./max. aanvulling** – Deze strategie gebruikt minimum- en maximumopslaglimieten om te bepalen wanneer locaties moeten worden aangevuld.</span><span class="sxs-lookup"><span data-stu-id="0c274-109">**Min/Max replenishment** – This strategy uses minimum and maximum stocking limits to determine when locations should be replenished.</span></span> <span data-ttu-id="0c274-110">De artikel- en locatiecriteria definiëren de voorraad die voor aanvulling wordt beoordeeld.</span><span class="sxs-lookup"><span data-stu-id="0c274-110">The item and location criteria define the inventory that is evaluated for replenishment.</span></span> <span data-ttu-id="0c274-111">Sjablonen voor Min/Max aanvulling zijn het belangrijkste mechanisme voor het onderhouden van optimale niveaus op orderverzamellocaties.</span><span class="sxs-lookup"><span data-stu-id="0c274-111">Min/Max replenishment templates are the primary mechanism for maintaining optimal levels in picking locations.</span></span> <span data-ttu-id="0c274-112">Om te garanderen dat voldoende voorraad voor orderverzamelen beschikbaar is om te voldoen aan wave-vraag, kunt u vraagaanvulling gebruiken als aanvulling tussen cycli voor Min/Max aanvulling.</span><span class="sxs-lookup"><span data-stu-id="0c274-112">To help guarantee that enough pick face inventory is available to meet wave demand, you can use demand replenishment as a supplement between Min/Max replenishment cycles.</span></span>
- <span data-ttu-id="0c274-113">**Vraag voor aanvulling laden** – Bij deze strategie wordt de vraag voor verschillende ladingen opgeteld en wordt het aanvullingswerk gemaakt dat is vereist om de relevante orderverzamellocaties te bevoorraden.</span><span class="sxs-lookup"><span data-stu-id="0c274-113">**Load demand replenishment** – This strategy sums the demand for several loads and creates the replenishment work that is required in order to stock the relevant picking locations.</span></span> <span data-ttu-id="0c274-114">Deze strategie helpt te garanderen dat de ladingen die zijn gemaakt kunnen worden verzameld in het magazijn nadat ze zijn vrijgegeven.</span><span class="sxs-lookup"><span data-stu-id="0c274-114">This strategy helps guarantee that the loads that are created can be picked in the warehouse after they are released.</span></span>
- <span data-ttu-id="0c274-115">**Directe aanvulling**: met deze strategie wordt voorraad aangevuld vóór het uitvoeren van een wave als toewijzing mislukt voor een locatie-instructieregel met een aanvullingssjabloon.</span><span class="sxs-lookup"><span data-stu-id="0c274-115">**Immediate replenishment** – This strategy replenishes inventory before a wave is run if allocation fails for a location directive line that has a replenishment template.</span></span> 

<span data-ttu-id="0c274-116">Alle vier de strategieën maken aanvullingswerk, op basis van een sjabloon voor aanvulling.</span><span class="sxs-lookup"><span data-stu-id="0c274-116">All four strategies create replenishment work, based on a replenishment template.</span></span>

## <a name="wave-demand-replenishment"></a><span data-ttu-id="0c274-117">Aanvulling voor wave-vraag</span><span class="sxs-lookup"><span data-stu-id="0c274-117">Wave demand replenishment</span></span>
<span data-ttu-id="0c274-118">Bij aanvulling voor wave-vraag wordt aanvullingswerk gemaakt op basis van de vraag als de vereiste hoeveelheid voor productieorders, kanbans, uitgaande orders of ladingen niet beschikbaar is wanneer een wave werk maakt.</span><span class="sxs-lookup"><span data-stu-id="0c274-118">Wave demand replenishment creates replenishment work, based on demand, if the quantity that is required for production orders, kanbans, outbound orders, or loads isn't available when a wave creates work.</span></span> <span data-ttu-id="0c274-119">De aanvullingssjabloon bevat informatie over de artikelcriteria, de maateenheid, de toename van de vraag en de locatie.</span><span class="sxs-lookup"><span data-stu-id="0c274-119">The replenishment template contains information about the item criteria, the unit of measure, the demand increment, and the location.</span></span> 

<span data-ttu-id="0c274-120">Locatie-instructies worden gebruikt om te bepalen welke locatie moet worden aangevuld.</span><span class="sxs-lookup"><span data-stu-id="0c274-120">Location directives are used to determine which location should be replenished.</span></span> <span data-ttu-id="0c274-121">U koppelt deze locatie-instructies aan de aanvullingssjabloon met behulp van het veld **Instructiecode**.</span><span class="sxs-lookup"><span data-stu-id="0c274-121">You link these location directives to the replenishment template by using the **Directive code** field.</span></span> <span data-ttu-id="0c274-122">Als het veld **Instructiecode** niet is ingesteld, worden query's gebruikt om te bepalen welke locatie-instructie moet worden gebruikt.</span><span class="sxs-lookup"><span data-stu-id="0c274-122">If the **Directive code** field isn't set, queries are used to determine which location directive should be used.</span></span> <span data-ttu-id="0c274-123">Houd er rekening mee dat als geen instructiecode is opgegeven in de aanvullingssjabloon en de locatie-instructie een instructiecode heeft, wordt de locatie-instructie genegeerd, zelfs als de query op de locatie-instructie correct is.</span><span class="sxs-lookup"><span data-stu-id="0c274-123">Note that if a directive code isn't specified in the replenishment template, and the location directive has a directive code, the location directive will be ignored, even if the query on the location directive is correct.</span></span> <span data-ttu-id="0c274-124">Locatie-instructies voor orderverzamelen worden gebruikt om te bepalen waar u voorraad voor de aanvulling vandaan moet halen.</span><span class="sxs-lookup"><span data-stu-id="0c274-124">Pick location directives are used to determine where to get inventory for the replenishment.</span></span> 

<span data-ttu-id="0c274-125">Naast een sjabloon maken, moet u enkele aanvullingsinstellingen opgeven in de wave-sjabloon.</span><span class="sxs-lookup"><span data-stu-id="0c274-125">In addition to creating a template, you must specify some replenishment settings in the wave template.</span></span> <span data-ttu-id="0c274-126">De wave-sjabloon moet een wave-stap voor aanvulling bevatten die alleen wordt uitgevoerd als toewijzing van een artikel mislukt.</span><span class="sxs-lookup"><span data-stu-id="0c274-126">The wave template should contain a wave step for replenishment that is run only if an item isn't successfully allocated.</span></span> <span data-ttu-id="0c274-127">Deze wave-stap voor aanvulling gebruikt een wave-stapcode om te bepalen welke aanvullingssjabloon moet worden gebruikt.</span><span class="sxs-lookup"><span data-stu-id="0c274-127">This replenishment wave step uses a wave step code to determine which replenishment template should be used.</span></span> <span data-ttu-id="0c274-128">U moet niet alleen een wave-stap voor aanvulling instellen, maar ook zorgen dat **Aanvullen** is geselecteerd in de sectie **Methoden** van de wave-sjabloon.</span><span class="sxs-lookup"><span data-stu-id="0c274-128">In addition to having a wave step for replenishment, you must make sure that **Replenish** is selected in the **Methods** section of the wave template.</span></span> 

<span data-ttu-id="0c274-129">De pagina **Aanvullingssjabloon** bevat een selectievakje **Gebruik van niet-gereserveerde hoeveelheid door waveaanvraag toestaan**.</span><span class="sxs-lookup"><span data-stu-id="0c274-129">The **Replenishment template** page includes an **Allow wave demand to use unreserved quantities** check box.</span></span> <span data-ttu-id="0c274-130">Schakel dit selectievakje in als u vraagaanvulling wilt toestaan om niet-gereserveerde hoeveelheden van werk af te trekken van werk dat is gegenereerd op basis van de geselecteerde aanvullingssjabloon.</span><span class="sxs-lookup"><span data-stu-id="0c274-130">Select this check box if demand replenishment should be able to deduct unreserved quantities from work that is generated from the selected replenishment template.</span></span> <span data-ttu-id="0c274-131">Als u sjablonen voor vraagaanvulling in staat wilt stellen om deze logica te gebruiken, moet u dit selectievakje inschakelen voor elke bestaande aanvullingssjabloon.</span><span class="sxs-lookup"><span data-stu-id="0c274-131">To enable demand replenishment templates to use this logic, select this check box for every existing replenishment template.</span></span> <span data-ttu-id="0c274-132">Wanneer de vraagaanvulling in het magazijn wordt geactiveerd, wordt de vraag afgetrokken van het bestaande aanvullingswerk dat niet-gereserveerde hoeveelheden heeft, als het werk afkomstig is uit aanvullingssjablonen waarbij het selectievakje **Gebruik van niet-gereserveerde hoeveelheid door waveaanvraag toestaan** is ingeschakeld.</span><span class="sxs-lookup"><span data-stu-id="0c274-132">When demand replenishment is triggered in the warehouse, it will deduct the demand from existing replenishment work that has unreserved quantities, if the work originates from replenishment templates where the **Allow wave demand to use unreserved quantities** check box is selected.</span></span>

<span data-ttu-id="0c274-133">Vraagaanvulling wordt ondersteund voor verkooporders, transferorders, productieorders en kanbans.</span><span class="sxs-lookup"><span data-stu-id="0c274-133">Demand replenishment is supported for sales orders, transfer orders, production orders, and kanbans.</span></span> 

## <a name="minmax-replenishment"></a><span data-ttu-id="0c274-134">Min/Max aanvulling</span><span class="sxs-lookup"><span data-stu-id="0c274-134">Min/Max replenishment</span></span>

<span data-ttu-id="0c274-141">Houd er rekening mee dat met de strategie van Min/Max-aanvulling geen lege locatie kan worden aangevuld tenzij de locatie is ingesteld als de vaste locatie voor het artikel.</span><span class="sxs-lookup"><span data-stu-id="0c274-141">Note that the Min/Max replenishment strategy can't replenish an empty location unless the location is set as the fixed location for the item.</span></span> <span data-ttu-id="0c274-142">Als de locatie die moet worden aangevuld geen vaste locatie is, kan het systeem niet bepalen welk artikel moet worden aangevuld.</span><span class="sxs-lookup"><span data-stu-id="0c274-142">If the location that must be replenished isn't a fixed location, the system can't determine which item should be replenished.</span></span> <span data-ttu-id="0c274-143">Daarom is ten minste enige voorhanden voorraad vereist voordat aanvulling plaatsvindt.</span><span class="sxs-lookup"><span data-stu-id="0c274-143">Therefore, at least some on-hand quantity is required before replenishment occurs.</span></span>
