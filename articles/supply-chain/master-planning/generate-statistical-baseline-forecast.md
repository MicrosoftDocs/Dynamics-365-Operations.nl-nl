# <a name="generate-a-statistical-baseline-forecast"></a><span data-ttu-id="60560-103">Een statistische basislijnprognose genereren</span><span class="sxs-lookup"><span data-stu-id="60560-103">Generate a statistical baseline forecast</span></span>

[!include [banner](../includes/banner.md)]

<span data-ttu-id="60560-104">Dit onderwerp bevat informatie over de parameters en de filters die in de berekening van de vraagprognose worden gebruikt.</span><span class="sxs-lookup"><span data-stu-id="60560-104">This topic provides information about the parameters and filters that are used in the calculation of demand forecasting.</span></span> 

<span data-ttu-id="60560-105">Wanneer u een basislijnprognose maakt, moet u eerst de parameters en filters opgeven die worden gebruikt in de berekening.</span><span class="sxs-lookup"><span data-stu-id="60560-105">When you create a baseline forecast, you must first specify the parameters and filters that are used in the calculation.</span></span> <span data-ttu-id="60560-106">U kunt bijvoorbeeld u een basislijnprognose maken die de vraag raamt op basis van transactiegegevens van het afgelopen jaar voor een specifiek bedrijf, voor de volgende maand, en voor een geselecteerde groep artikelen.</span><span class="sxs-lookup"><span data-stu-id="60560-106">For example, you can create a baseline forecast that estimates demand based on transaction data from the past year for a specific company, for the coming month, and for a selected group of items.</span></span> 

<span data-ttu-id="60560-107">Ga om een vraagprognose te genereren naar **Hoofdplanning &gt; Prognose &gt; Vraagprognose &gt; Statistische basislijnprognose maken**.</span><span class="sxs-lookup"><span data-stu-id="60560-107">To generate a demand forecast, go to **Master planning &gt; Forecasting &gt; Demand forecasting &gt; Generate statistical baseline forecast**.</span></span> 

<span data-ttu-id="60560-108">De prognoseverzameling kan worden geselecteerd in de prognosegeneratietijd.</span><span class="sxs-lookup"><span data-stu-id="60560-108">The forecast bucket can be selected at forecast generation time.</span></span> <span data-ttu-id="60560-109">De beschikbare waarden zijn Dag, Week en Maand.</span><span class="sxs-lookup"><span data-stu-id="60560-109">The available values are: Day, Week, and Month.</span></span> 

<span data-ttu-id="60560-110">Het aantal verzamelingen waarvoor u een prognose wilt genereren is ingesteld in het veld **Prognoseperiode**.</span><span class="sxs-lookup"><span data-stu-id="60560-110">The number of buckets to generate a forecast for is set in the **Forecast horizon** field.</span></span> 

<span data-ttu-id="60560-111">Wanneer de prognosestrategie is ingesteld **Kopieer over historische vraag**, wordt het einde van de historische periode genegeerd.</span><span class="sxs-lookup"><span data-stu-id="60560-111">When the forecast strategy is set to **Copy over historical demand**, the end of the historical horizon is ignored.</span></span> <span data-ttu-id="60560-112">Het aantal prognoseverzamelingen dat is opgegeven in het veld **Prognoseperiode**, wordt gekopieerd naar de prognosevraag, te beginnen met de datum die is ingesteld in het veld **Begindatum** onder **Historische periode**.</span><span class="sxs-lookup"><span data-stu-id="60560-112">The system copies the number of buckets specified in the **Forecast horizon** field to the forecast demand, starting from the date set in the **From date** field under **Historical horizon**.</span></span> <span data-ttu-id="60560-113">Door historische vraag van een bepaalde datum voorwaarts te kopiëren kunnen productieplanners het plan voor het kwartaal op twee manieren maken:</span><span class="sxs-lookup"><span data-stu-id="60560-113">By copying historical demand from a certain date forward, production planners can make the plan for the next quarter in two ways:</span></span>

-   <span data-ttu-id="60560-114">Door de vraag van hetzelfde kwartaal van het vorige jaar te kopiëren.</span><span class="sxs-lookup"><span data-stu-id="60560-114">By copying the demand from the same quarter last year.</span></span>
-   <span data-ttu-id="60560-115">Door de vraag van het voorafgaande kwartaal te kopiëren.</span><span class="sxs-lookup"><span data-stu-id="60560-115">By copying the demand from the previous quarter.</span></span>

<span data-ttu-id="60560-116">Om verwarring in de productieplannen te vermijden, kan een bepaald aantal prognoseverzamelingen worden vergrendeld.</span><span class="sxs-lookup"><span data-stu-id="60560-116">To prevent confusion in the production plans, a certain number of forecast buckets can be frozen.</span></span> <span data-ttu-id="60560-117">Dit aantal wordt ingesteld in het veld **Blokkering van de tijdlimiet**.</span><span class="sxs-lookup"><span data-stu-id="60560-117">This number is set in the **Freeze time fence** field.</span></span> <span data-ttu-id="60560-118">Op de **Aangepaste vraagprognose** pagina, worden de cellen voor de stilgezette verzamelingen uitgeschakeld, om een grafische indicatie te geven dat deze waarden niet moeten worden gewijzigd.</span><span class="sxs-lookup"><span data-stu-id="60560-118">On the **Adjusted demand forecast** page, the cells for the frozen buckets are disabled, to give a visual indication that those values should not be changed.</span></span> 

<span data-ttu-id="60560-119">De begindatum voor de basislijnvraagprognose hoeft niet de huidige datum of een datum in de toekomst te zijn.</span><span class="sxs-lookup"><span data-stu-id="60560-119">The start date for the baseline demand forecast doesn’t have to be the current date or a date in the future.</span></span> <span data-ttu-id="60560-120">Om een andere begindatum in te stellen, gebruikt u het veld **De begindatum van de basislijnprognose - Begindatum**.</span><span class="sxs-lookup"><span data-stu-id="60560-120">To set a different start date, use the **Baseline forecast start date - From date** field.</span></span> <span data-ttu-id="60560-121">In juni kunnen gebruikers bijvoorbeeld een prognose voor het volgende jaar genereren.</span><span class="sxs-lookup"><span data-stu-id="60560-121">For example, in June, users can generate a forecast for the next year.</span></span> <span data-ttu-id="60560-122">Omdat de prognoseverzamelingen tussen het einde van historische vraag en het begin van de basislijn ontbreken, zijn de voorspellingen mogelijk niet nauwkeurig.</span><span class="sxs-lookup"><span data-stu-id="60560-122">Because the forecast buckets between the end of historical demand and the start of the baseline are missing, the predictions might not be accurate.</span></span> <span data-ttu-id="60560-123">Als u de vraagprognoseservice gebruikt, zijn er vier manieren waarop u de lacunes kunt invullen.</span><span class="sxs-lookup"><span data-stu-id="60560-123">If you are using the Demand forecasting service, there are four ways in which you can fill in the missing gaps.</span></span> <span data-ttu-id="60560-124">U kunt de gewenste methode selecteren door de parameter MISSING\_VALUE\_SUBSTITUTION in te stellen op de pagina **Parameters voor vraagprognose**.</span><span class="sxs-lookup"><span data-stu-id="60560-124">You can choose the method that you want by setting the MISSING\_VALUE\_SUBSTITUTION parameter on the **Demand forecasting parameters** page.</span></span> 

> [!NOTE]
> <span data-ttu-id="60560-125">Ontbrekende waardevervanging werkt alleen voor gaten in gegevens tussen de begin- en eind datum van historische gegevens.</span><span class="sxs-lookup"><span data-stu-id="60560-125">Missing value substitution only works for the gaps in data in between the start and end dates for historical data.</span></span> <span data-ttu-id="60560-126">Er worden geen gegevens ingevuld voor of na het laatste fysieke gegevenspunt, het fungeert alleen als extrapolatie tussen werkelijk bestaande gegevenspunten.</span><span class="sxs-lookup"><span data-stu-id="60560-126">It will not fill in data before or after the last physical data point, it only acts as extrapolation between actual existing data points.</span></span> 

<span data-ttu-id="60560-127">Het veld **Begindatum van basislijnprognose** - **Begindatum** moet worden ingesteld aan het begin van een prognoseverzameling, bijvoorbeeld in de Verenigde Staten op een zondag als de prognoseverzameling de week is.</span><span class="sxs-lookup"><span data-stu-id="60560-127">The **Baseline forecast start date** - **From date** field has to be set to the beginning of a forecast bucket, for example, in the United States, a Sunday if the forecasting bucket is the week.</span></span> <span data-ttu-id="60560-128">Het veld **Begindatum van basislijnprognose** - **Begindatum** wordt automatisch aan het begin van een prognoseverzameling aangepast.</span><span class="sxs-lookup"><span data-stu-id="60560-128">The system automatically adjusts the **Baseline forecast start date** - **From date** field to match the beginning of a forecast bucket.</span></span> 

<span data-ttu-id="60560-129">Het veld **Begindatum van basislijnprognose** - **Begindatum** kan worden ingesteld op een datum in het verleden.</span><span class="sxs-lookup"><span data-stu-id="60560-129">The **Baseline forecast start date** - **From date** field can be set to a date in in the past.</span></span> <span data-ttu-id="60560-130">Met andere woorden, het is mogelijk om een vraagprognose in het verleden te genereren.</span><span class="sxs-lookup"><span data-stu-id="60560-130">In other words, it is possible to generate a demand forecast in the past.</span></span> <span data-ttu-id="60560-131">Dit is handig, omdat het gebruikers in staat stelt de parameters van de prognoseservice aan te passen zodat de statistische prognose die in het verleden is gegenereerd overeenkomt met de werkelijke historische vraag.</span><span class="sxs-lookup"><span data-stu-id="60560-131">This is useful, because it lets users adjust the forecast service parameters so that the statistical forecast generated in the past matches the actual historical demand.</span></span> <span data-ttu-id="60560-132">Gebruikers kunnen vervolgens deze parameterinstellingen blijven gebruiken om een statistische basislijnprognose te genereren voor de toekomst.</span><span class="sxs-lookup"><span data-stu-id="60560-132">Users can then continue using these parameter settings to generate a statistical baseline forecast for the future.</span></span> 

<span data-ttu-id="60560-133">Handmatige correcties die bij vorige iteraties van vraagprognoses zijn aangebracht, kunnen automatisch worden toegepast op de nieuwe basislijnprognose als het selectievakje **Handmatige correcties overbrengen naar de vraagprognose** is ingeschakeld.</span><span class="sxs-lookup"><span data-stu-id="60560-133">Manual adjustments made in previous demand forecasting iterations can be automatically applied to the new baseline forecast if the **Transfer manual adjustments to the demand forecas** t check box is selected.</span></span> <span data-ttu-id="60560-134">Als het selectievakje is uitgeschakeld, worden handmatige aanpassingen niet toegevoegd aan de basislijnprognose, maar ze worden niet verwijderd.</span><span class="sxs-lookup"><span data-stu-id="60560-134">If the check box is cleared, the manual adjustments are not added to the baseline forecast – but they are not deleted.</span></span> <span data-ttu-id="60560-135">Handmatige correcties die in een prognose zijn aangebracht, kunnen alleen op het moment van importeren van de prognose worden verwijderd door het selectievakje **De handmatige correcties opslaan die in de basislijnvraagprognose zijn gemaakt** te wissen.</span><span class="sxs-lookup"><span data-stu-id="60560-135">Manual adjustments made to a forecast can be deleted only at forecast import time, by clearing the **Save the manual adjustments made to the baseline demand forecast** check box.</span></span> <span data-ttu-id="60560-136">Handmatige aanpassingen worden opgeslagen op het moment van autorisatie.</span><span class="sxs-lookup"><span data-stu-id="60560-136">Manual adjustments are saved at authorization time.</span></span> <span data-ttu-id="60560-137">Daarom gaan de wijzigingen verloren als een gebruiker handmatig aanpassingen uitvoert in de prognose, maar de prognose niet opnieuw autoriseert in Supply Chain Management.</span><span class="sxs-lookup"><span data-stu-id="60560-137">Therefore, if a user makes manual adjustments to the forecast, but doesn’t authorize the forecast back to Supply Chain Management, the changes are lost.</span></span> <span data-ttu-id="60560-138">Zie [De gecorrigeerde prognose autoriseren](authorize-adjusted-forecast.md) voor meer informatie over handmatige correcties en hoe deze werken.</span><span class="sxs-lookup"><span data-stu-id="60560-138">For more information about manual adjustments and how they work, see [Authorize an adjusted forecast](authorize-adjusted-forecast.md).</span></span> 

<span data-ttu-id="60560-139">Het genereren van een vraagprognose kan een naam en opmerkingen hebben om gebruikers te helpen de gegenereerde prognose te identificeren.</span><span class="sxs-lookup"><span data-stu-id="60560-139">A demand forecast generation can have a name and comments to help users identify the forecast that has been generated.</span></span> <span data-ttu-id="60560-140">Deze waarden zijn zichtbaar in de historie voor het genereren van prognoses op de pagina **Historie van genereren van statistische basislijnprognose**.</span><span class="sxs-lookup"><span data-stu-id="60560-140">These values are visible in forecast generation history on the **Statistical baseline forecast generation history** page.</span></span> 

<span data-ttu-id="60560-141">De intercompany-planninggroep, de artikeltoewijzingssleutels, en andere filters kunnen in de prognosegeneratietijd worden toegepast.</span><span class="sxs-lookup"><span data-stu-id="60560-141">The intercompany planning group, item allocation keys, and other filters can be applied at forecast generation time.</span></span> <span data-ttu-id="60560-142">Deze kunnen worden gebruikt om de prestaties te verbeteren of de gegevens op te splitsen in werkbare brokken.</span><span class="sxs-lookup"><span data-stu-id="60560-142">These can be used to improve performance or to split the data into manageable chunks.</span></span> <span data-ttu-id="60560-143">Er wordt echter geen vraagprognose gegenereerd voor de leden van een artikeltoewijzingssleutel die niet zijn gekoppeld aan een intercompany-planningsgroep, zelfs als de artikeltoewijzingssleutel in de query wordt geselecteerd.</span><span class="sxs-lookup"><span data-stu-id="60560-143">However, note that a demand forecast is not generated for the members of any item allocation key that is not associated with an intercompany planning group, even if the item allocation key is selected in the query.</span></span> 

> [!TIP]
> <span data-ttu-id="60560-144">Soms kunnen gebruikers fouten krijgen tijdens het genereren van een vraagprognose, of wordt het maken van de prognose voltooid zonder sessielogboek.</span><span class="sxs-lookup"><span data-stu-id="60560-144">Sometimes users might receive errors while generating a demand forecast, or forecast generation is completed with no session log.</span></span> <span data-ttu-id="60560-145">Dit kan gebeuren vanwege resterende gegevens in de query die eerder werd gebruikt voor het genereren van een prognose.</span><span class="sxs-lookup"><span data-stu-id="60560-145">This can happen because of leftover data in the query that was previously used for forecast generation.</span></span> <span data-ttu-id="60560-146">U kunt dit probleem oplossen door te klikken op **Selecteren** om de pagina **Query** te openen, op **Opnieuw instellen** te selecteren en vervolgens de basislijnprognose opnieuw te genereren.</span><span class="sxs-lookup"><span data-stu-id="60560-146">To resolve this issue, click **Select** to open the **Query** page, select **Reset**, and then regenerate the baseline forecast.</span></span> 

<span data-ttu-id="60560-147">Als de prognose niet wordt gegenereerd voor een grote set artikelen, maar bijvoorbeeld voor één artikel of artikeltoewijzingssleutel tegelijk, kunt u om betere prestaties te krijgen het selectievakje **Aanvraagreactiemodus gebruiken** op het tabblad **Hoofdplanning - Instellingen - Vraagprognose** - **Parameters voor vraagprognose - Azure Machine Learning** inschakelen.</span><span class="sxs-lookup"><span data-stu-id="60560-147">If the forecast is not generated for a big set of items, but, for example, for one item or one item allocation key at a time, then in order to get better performance, you can select the **Use request response mode** check box on the **Master planning - Setup - Demand forecasting** - **Demand forecasting parameters - Azure Machine Learning** tab.</span></span>

> [!NOTE]
> <span data-ttu-id="60560-148">Een potentieel vlakke prognose kan het gevolg zijn van de historische gegevens die een langere historische periode moeten omvatten (minimaal drie tijdsperioden om patronen te kunnen zien, bijvoorbeeld 3 jaar met maandelijkse prognose).</span><span class="sxs-lookup"><span data-stu-id="60560-148">A potentially flat looking forecast can be due to the historical data that has to be of a longer historical timeframe (a minimum 3 of time periods in order to pick out patterns, such as 3 years with monthly forecast).</span></span> <span data-ttu-id="60560-149">U kunt proberen de nauwkeurigheid van het tijdsbereik te wijzigen of het tijdsbereik te vergroten om een beter resultaat te krijgen.</span><span class="sxs-lookup"><span data-stu-id="60560-149">To get a better result, you can try changing the granularity of the time range or increase the time range.</span></span>
