# <a name="demand-forecasting-setup"></a><span data-ttu-id="ff142-103">Instelling van vraagprognose</span><span class="sxs-lookup"><span data-stu-id="ff142-103">Demand forecasting setup</span></span>

[!include[banner](../includes/banner.md)]


<span data-ttu-id="ff142-104">In dit onderwerp worden de insteltaken beschreven die u moet uitvoeren ter voorbereiding op de vraagprognose.</span><span class="sxs-lookup"><span data-stu-id="ff142-104">This topic describes the setup tasks that you must perform to prepare for demand forecasting.</span></span>

## <a name="item-allocation-key"></a><span data-ttu-id="ff142-106">Artikeltoewijzingssleutel</span><span class="sxs-lookup"><span data-stu-id="ff142-106">Item allocation key</span></span>
<span data-ttu-id="ff142-107">Een vraagprognose wordt alleen berekend voor een artikel en de dimensies ervan als het artikel onderdeel uitmaakt van een artikeltoewijzingssleutel.</span><span class="sxs-lookup"><span data-stu-id="ff142-107">A demand forecast is calculated for an item and its dimensions only if the item is part of an item allocation key.</span></span> <span data-ttu-id="ff142-108">Deze regel wordt afgedwongen om grote aantallen artikelen te groeperen, zodat vraagprognoses sneller kunnen worden gemaakt.</span><span class="sxs-lookup"><span data-stu-id="ff142-108">This rule is enforced to group large numbers of items, so that demand forecasts can be created more quickly.</span></span> <span data-ttu-id="ff142-109">Het sleutelpercentage voor artikeltoewijzing wordt genegeerd wanneer vraagprognoses worden gegenereerd.</span><span class="sxs-lookup"><span data-stu-id="ff142-109">The item allocation key percentage is ignored when demand forecasts are generated.</span></span> <span data-ttu-id="ff142-110">Prognoses worden gemaakt op basis van alleen historische gegevens.</span><span class="sxs-lookup"><span data-stu-id="ff142-110">Forecasts are created based on historical data only.</span></span> 

<span data-ttu-id="ff142-111">Een artikel en de dimensies ervan moeten onderdeel zijn van slechts één artikeltoewijzingssleutel als de artikeltoewijzingssleutel wordt gebruikt tijdens het maken van prognoses.</span><span class="sxs-lookup"><span data-stu-id="ff142-111">An item and its dimensions must be part of only one item allocation key if the item allocation key is used during forecast creation.</span></span> 

<span data-ttu-id="ff142-112">Als u een voorraadeenheid (SKU) aan een artikeltoewijzingssleutel wilt toevoegen, gaat u naar **Hoofdplanning** &gt; **Instellen** &gt; **Vraagprognose** &gt; **Artikeltoewijzingssleutels**.</span><span class="sxs-lookup"><span data-stu-id="ff142-112">To add a stock keeping unit (SKU) to an item allocation key, go to **Master planning** &gt; **Setup** &gt; **Demand forecasting** &gt; **Item allocation keys**.</span></span> <span data-ttu-id="ff142-113">Gebruik de pagina **Artikelen toewijzen** om een artikel aan een toewijzingssleutel toe te wijzen.</span><span class="sxs-lookup"><span data-stu-id="ff142-113">Use the **Assign items** page to assign an item to an allocation key.</span></span>

## <a name="intercompany-planning-groups"></a><span data-ttu-id="ff142-114">Intercompany-planningsgroepen</span><span class="sxs-lookup"><span data-stu-id="ff142-114">Intercompany planning groups</span></span>
<span data-ttu-id="ff142-115">Vraagprognoses genereren cross-company prognoses.</span><span class="sxs-lookup"><span data-stu-id="ff142-115">Demand forecasting generates cross-company forecasts.</span></span> <span data-ttu-id="ff142-116">In Microsoft Dynamics 365 for Finance and Operations worden bedrijven die samen worden gepland, gegroepeerd in één intercompany-planningsgroep.</span><span class="sxs-lookup"><span data-stu-id="ff142-116">In Microsoft Dynamics 365 for Finance and Operations, companies that are planned together are grouped into one intercompany planning group.</span></span> <span data-ttu-id="ff142-117">Om per bedrijf op te geven welke artikeltoewijzingssleutels moeten worden meegenomen voor de vraagprognoses , koppelt u een artikeltoewijzingssleutel met het lid van de intercompany-planningsgroep door naar **Hoofdplanning** &gt; **Instellen** &gt; **Intercompany-planningsgroepen** te gaan.</span><span class="sxs-lookup"><span data-stu-id="ff142-117">To specify, per company, which item allocation keys should be considered for demand forecasting, associate an item allocation key with the intercompany planning group member by going to **Master planning** &gt; **Setup** &gt; **Intercompany planning groups**.</span></span> 

<span data-ttu-id="ff142-118">Als geen artikeltoewijzingssleutels aan leden van intercompany-planningsgroepen worden toegewezen, wordt standaard een vraagprognose berekend voor alle artikelen die worden toegewezen aan alle artikeltoewijzingssleutels vanuit alle Finance and Operations-bedrijven.</span><span class="sxs-lookup"><span data-stu-id="ff142-118">By default, if no item allocation keys are assigned to intercompany planning group members, a demand forecast is calculated for all items that are assigned to all item allocation keys from all Finance and Operations companies.</span></span> <span data-ttu-id="ff142-119">Extra filteropties voor bedrijven en artikeltoewijzingssleutels zijn beschikbaar op de pagina **Statistische basislijnprognose maken**.</span><span class="sxs-lookup"><span data-stu-id="ff142-119">Additional filtering options for companies and item allocation keys are available on the **Generate statistical baseline forecast** page.</span></span> 

<span data-ttu-id="ff142-120">Controleer het aantal artikelen dat wordt verwacht.</span><span class="sxs-lookup"><span data-stu-id="ff142-120">Review the number of items that are forecasted.</span></span> <span data-ttu-id="ff142-121">Onnodige artikelen kunnen verhoogde kosten veroorzaken wanneer u Microsoft Azure Machine Learning gebruikt.</span><span class="sxs-lookup"><span data-stu-id="ff142-121">Unnecessary items might cause increased costs when you use Microsoft Azure Machine Learning.</span></span>

## <a name="demand-forecasting-parameters"></a><span data-ttu-id="ff142-122">Parameters voor vraagprognose</span><span class="sxs-lookup"><span data-stu-id="ff142-122">Demand forecasting parameters</span></span>
<span data-ttu-id="ff142-123">Als u vraagprognoseparameters wilt instellen, gaat u naar **Hoofdplanning** &gt; **Instellen** &gt; **Parameters voor vraagprognose**.</span><span class="sxs-lookup"><span data-stu-id="ff142-123">To set up demand forecasting parameters, go to **Master planning** &gt; **Setup** &gt; **Demand forecasting parameters**.</span></span> <span data-ttu-id="ff142-124">Omdat vraagprognose cross-company wordt uitgevoerd, is de instelling globaal.</span><span class="sxs-lookup"><span data-stu-id="ff142-124">Because demand forecasting runs cross-company, the setup is global.</span></span> <span data-ttu-id="ff142-125">Met andere woorden, geldt de instelling voor alle bedrijven.</span><span class="sxs-lookup"><span data-stu-id="ff142-125">In other words, the setup applies to all companies.</span></span> 

<span data-ttu-id="ff142-126">Vraagprognoses genereren de prognose in hoeveelheden.</span><span class="sxs-lookup"><span data-stu-id="ff142-126">Demand forecasting generates the forecast in quantities.</span></span> <span data-ttu-id="ff142-127">Daarom moet de maateenheid waarin de hoeveelheid moet worden uitgedrukt worden opgegeven in het veld **Vraagprognose-eenheid**.</span><span <span data-ttu-id="ff142-132">Vraagprognoses kunnen worden gebruikt voor prognoses van zowel afhankelijke als onafhankelijke vraag.</span><span class="sxs-lookup"><span data-stu-id="ff142-132">Demand forecasting can be used to forecast both dependent and independent demand.</span></span> <span data-ttu-id="ff142-133">Als bijvoorbeeld alleen het selectievakje **Verkooporder** is ingeschakeld en als alle artikelen die voor de vraagprognose worden beschouwd, artikelen zijn die worden verkocht, berekent het systeem onafhankelijke vraag.</span><span class="sxs-lookup"><span data-stu-id="ff142-133">For example, if only the **Sales order** check box is selected, and if all the items that are considered for demand forecasting are items that are sold, the system calculates independent demand.</span></span> <span data-ttu-id="ff142-134">Kritieke subcomponenten kunnen echter worden toegevoegd aan artikeltoewijzingssleutels en worden opgenomen in vraagprognoses.</span><span class="sxs-lookup"><span data-stu-id="ff142-134">However, critical subcomponents can be added to item allocation keys and included in demand forecasting.</span></span> <span data-ttu-id="ff142-135">In dit geval, als het selectievakje **Productieregel** is ingeschakeld, wordt een afhankelijke prognose berekend.</span><span class="sxs-lookup"><span data-stu-id="ff142-135">In this case, if the **Production line** check box is selected, a dependent forecast is calculated.</span></span>

<span data-ttu-id="ff142-140">Door te klikken op **Prognosedimensies** in het linkerdeelvenster van de pagina **Parameters voor vraagprognose** kunt u de set prognosedimensies selecteren om te gebruiken wanneer de vraagprognose wordt gegenereerd.</span><span class="sxs-lookup"><span data-stu-id="ff142-140">By clicking **Forecast dimensions** in the left pane of the **Demand forecasting parameters** page, you can also select the set of forecast dimensions to use when the demand forecast is generated.</span></span> <span data-ttu-id="ff142-141">Een prognosedimensie geeft het detailniveau aan waarvoor de prognose is gedefinieerd.</span><span class="sxs-lookup"><span data-stu-id="ff142-141">A forecast dimension indicates the level of detail that the forecast is defined for.</span></span> <span data-ttu-id="ff142-142">Bedrijf, site en artikeltoewijzingssleutel zijn verplichte prognosedimensies, maar u kunt prognoses ook genereren op het niveau van het magazijn, de voorraadstatus, de klantgroep, de klantrekening, het land/de regio, de status, het artikel en alle artikeldimensieniveaus.</span><span class="sxs-lookup"><span data-stu-id="ff142-142">Company, site, and item allocation key are mandatory forecast dimensions, but you can also generate forecasts at the warehouse, inventory status, customer group, customer account, country/region, state, and item plus all item dimension levels.</span></span> 

<span data-ttu-id="ff142-143">U kunt op elk moment prognosedimensies toevoegen aan de lijst met dimensies die voor vraagprognoses worden gebruikt.</span><span class="sxs-lookup"><span data-stu-id="ff142-143">At any point, you can add forecast dimensions to the list of dimensions that are used for demand forecasting.</span></span> <span data-ttu-id="ff142-144">U kunt ook eventueel prognosedimensie uit de lijst verwijderen.</span><span class="sxs-lookup"><span data-stu-id="ff142-144">You can also remove forecast dimensions from the list.</span></span> <span data-ttu-id="ff142-145">Handmatige correcties gaan echter verloren als u een prognosedimensie toevoegt of verwijdert.</span><span class="sxs-lookup"><span data-stu-id="ff142-145">However, manual adjustments are lost if you add or remove a forecast dimension.</span></span>
