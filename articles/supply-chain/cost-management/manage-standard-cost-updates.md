# <a name="manage-standard-cost-updates"></a><span data-ttu-id="d4352-103">Updates voor standaardkosten beheren</span><span class="sxs-lookup"><span data-stu-id="d4352-103">Manage standard cost updates</span></span>

[!include [banner](../includes/banner.md)]

<span data-ttu-id="d4352-104">Updates van standaardkostengegevens kunnen worden beheerd met twee verschillende benaderingen: de benadering met één versie en de benadering met twee versies.</span><span class="sxs-lookup"><span data-stu-id="d4352-104">Updates to standard cost data can be managed by using two different approaches - the one-version approach or the two-version approach.</span></span> 

<span data-ttu-id="d4352-105">Bij de benadering met één versie wordt één kostprijsberekeningsversie gebruikt die alle kostenregistraties bevat.</span><span class="sxs-lookup"><span data-stu-id="d4352-105">The one-version approach uses a single costing version that contains all cost records.</span></span> <span data-ttu-id="d4352-106">Deze registraties omvatten de oorspronkelijke koten en alle kostenupdates.</span><span class="sxs-lookup"><span data-stu-id="d4352-106">These records include the original costs and all cost updates.</span></span>

<span data-ttu-id="d4352-107">Bij de methode met twee versies wordt één versie gebruikt die registraties van de oorspronkelijke kosten bevat en een tweede versie die registraties alle van kostenupdates bevat.</span><span class="sxs-lookup"><span data-stu-id="d4352-107">The two-version approach uses one version that contains records of the original costs and a second version that contains records of all cost updates.</span></span> <span data-ttu-id="d4352-108">Het belangrijkste voordeel van de methode met twee versies is de duidelijke afbakening en tracering van kostenupdates in een aparte kostprijsberekeningsversie, zonder dat dit gevolgen heeft voor de oorspronkelijke kostprijsberekeningsversie.</span><span class="sxs-lookup"><span data-stu-id="d4352-108">A primary advantage of the two-version approach is the clear delineation and tracking of cost updates in a separate costing version, without affecting the original costing version.</span></span> <span data-ttu-id="d4352-109">De methode met twee versies kan worden gebruikt om meerdere incrementele updates te identificeren, waarbij elke incrementele update een aparte kostprijsberekeningsversie heeft die de incrementele kostenrecords bevat.</span><span class="sxs-lookup"><span data-stu-id="d4352-109">The two-version approach can be used to identify multiple incremental updates, where each incremental update has a separate costing version that contains the incremental cost records.</span></span> 

<span data-ttu-id="d4352-110">**Voorbeeld:**</span><span class="sxs-lookup"><span data-stu-id="d4352-110">**Example**</span></span> 

<span data-ttu-id="d4352-111">Het volgende voorbeeld illustreert hoe de methoden met één versie en met twee versies kunnen worden gebruikt voor het bijwerken van standaardkosten in een productieomgeving.</span><span class="sxs-lookup"><span data-stu-id="d4352-111">The following example illustrates how the one-version and two-version approaches can be used for updating standard costs in a manufacturing environment.</span></span> <span data-ttu-id="d4352-112">Bijvoorbeeld updates die nieuwe artikelen of foutcorrecties weerspiegelen.</span><span class="sxs-lookup"><span data-stu-id="d4352-112">For example, updates that reflect new items or error corrections.</span></span> <span data-ttu-id="d4352-113">Ga ervanuit dat één kostprijsberekeningsversie de standaardkosten voor het huidige jaar voorstelt.</span><span class="sxs-lookup"><span data-stu-id="d4352-113">Assume that a single costing version represents the standard costs for the current year.</span></span> <span data-ttu-id="d4352-114">De id voor deze versie is 2016-STD.</span><span class="sxs-lookup"><span data-stu-id="d4352-114">The identifier for this version is 2016-STD.</span></span> <span data-ttu-id="d4352-115">Versie 2016-STD bevat de huidige actieve kosten voor alle artikelen.</span><span class="sxs-lookup"><span data-stu-id="d4352-115">Version 2016-STD contains the current active costs for all items.</span></span> <span data-ttu-id="d4352-116">Bovendien bevat ze alle route-gerelateerde kostencategorieën en overheadberekeningsformules die aan het begin van het jaar 2016 bekend waren.</span><span class="sxs-lookup"><span data-stu-id="d4352-116">Additionally, it contains all routing-related cost categories and overhead calculation formulas that were known at the start of the year 2016.</span></span> <span data-ttu-id="d4352-117">2016-STD is de oorspronkelijke kostprijsberekeningsversie.</span><span class="sxs-lookup"><span data-stu-id="d4352-117">2016-STD is the original costing version.</span></span>

2016-STD will be untouched.</span></span> <span data-ttu-id="d4352-130">Bij de methode met twee kostprijsberekeningsversies worden updates voorkomen door blokkeringsbeleid voor de oorspronkelijke
