# <a name="ledger-allocation-rules"></a><span data-ttu-id="e3708-104">Grootboektoewijzingsregels</span><span class="sxs-lookup"><span data-stu-id="e3708-104">Ledger allocation rules</span></span>

[!include [banner](../includes/banner.md)]

<span data-ttu-id="e3708-105">Dit artikel bevat informatie over grootboektoewijzingsregels.</span><span class="sxs-lookup"><span data-stu-id="e3708-105">This article provides information about ledger allocation rules.</span></span> <span data-ttu-id="e3708-106">Het beschrijft de verschillende onderdelen van deze toewijzingsregels en toewijzingsmethoden die hiervoor kunnen worden gebruikt.</span><span class="sxs-lookup"><span data-stu-id="e3708-106">It describes the various components of these allocation rules and the allocation methods that can be used for them.</span></span>

<span data-ttu-id="e3708-107">Grootboektoewijzingsregels worden gebruikt om toewijzingsjournalen en journaalregels automatisch te berekenen en te genereren voor de toewijzing van grootboeksaldi of vaste bedragen.</span><span class="sxs-lookup"><span data-stu-id="e3708-107">Ledger allocation rules are used to automatically calculate and generate allocation journals and account entries for the allocation of ledger balances or fixed amounts.</span></span> <span data-ttu-id="e3708-108">Toewijzingsmethoden kunnen variabel of vast zijn.</span><span class="sxs-lookup"><span data-stu-id="e3708-108">Allocation methods can be variable or fixed.</span></span> <span data-ttu-id="e3708-109">De volgende toewijzingsmethoden kunnen voor grootboektoewijzingsregels worden gebruikt:</span><span class="sxs-lookup"><span data-stu-id="e3708-109">The following allocation methods can be used for ledger allocation rules:</span></span>

-   <span data-ttu-id="e3708-110">**Basis** - Deze variabele methode wordt gebruikt wanneer de toewijzing afhankelijk is van het werkelijke grootboeksaldo, gebaseerd op filtercriteria.</span><span class="sxs-lookup"><span data-stu-id="e3708-110">**Basis** – This variable method is used when the allocation depends on the actual ledger balance, based on filter criteria.</span></span> <span data-ttu-id="e3708-111">Reclameonkosten bijvoorbeeld kunnen worden toegewezen op basis van de verkoop van elke afdeling in verhouding tot de totale afdelingsverkoop.</span><span class="sxs-lookup"><span data-stu-id="e3708-111">For example, advertising expenses can be allocated based on each department's sales in proportion to the total departmental sales.</span></span>
-   <span data-ttu-id="e3708-112">**Vast percentage** en **Vast gewicht** - Voor deze methoden wordt het toewijzingspercentage of -gewicht direct gedefinieerd voor de regel.</span><span class="sxs-lookup"><span data-stu-id="e3708-112">**Fixed percentage** and **Fixed weight** – For these methods, the allocation percentage or weight is defined directly for the rule.</span></span> <span data-ttu-id="e3708-113">Reclameonkosten bijvoorbeeld kunnen worden toegewezen zodat Afdeling A 70 procent van de reclameonkosten ontvangt en Afdeling B 30 procent.</span><span class="sxs-lookup"><span data-stu-id="e3708-113">For example, advertising expenses can be allocated so that Department A receives 70 percent of the advertising expense and Department B receives 30 percent.</span></span>
-   <span data-ttu-id="e3708-114">**Evenredig** - Deze methode verdeelt het bedrag evenredig over elk gedefinieerd doel.</span><span class="sxs-lookup"><span data-stu-id="e3708-114">**Equally** – This method distributes the amount equally to each defined destination.</span></span> <span data-ttu-id="e3708-115">Stel dat er doelen zijn gedefinieerd voor Afdeling A en Afdeling B, dan kunnen de reclameonkosten zo worden toegewezen dat zowel Afdeling A als Afdeling B 50 procent van de reclameonkosten ontvangen.</span><span class="sxs-lookup"><span data-stu-id="e3708-115">For example, if destinations are defined for Department A and Department B, advertising expenses can be allocated so that both Department A and Department B receive 50 percent of the advertising expense.</span></span>

<span data-ttu-id="e3708-116">Als Basis als toewijzingsmethode voor een toewijzingsregel wordt gebruikt, moet u ook afzonderlijke basisregels voor grootboektoewijzing definiëren.</span><span class="sxs-lookup"><span data-stu-id="e3708-116">If Basis is used as the allocation method for an allocation rule, you must also define separate ledger allocation basis rules.</span></span> <span data-ttu-id="e3708-117">Gebruikers kunnen met het proces "Toewijzingsaanvraag verwerken" de grootboektoewijzingsregel verwerken en de resulterende toewijzingsjournaalregels bekijken voordat ze de berekende toewijzingen boeken of verwijderen.</span><span class="sxs-lookup"><span data-stu-id="e3708-117">The "Process allocation request" process lets users process the ledger allocation rule and preview the resulting allocation journal entries before they either post or delete the calculated allocations.</span></span>

## <a name="components-of-ledger-allocation-rules"></a><span data-ttu-id="e3708-118">Onderdelen van grootboektoewijzingsregels</span><span class="sxs-lookup"><span data-stu-id="e3708-118">Components of ledger allocation rules</span></span>
<span data-ttu-id="e3708-119">Elke toewijzingsregel bestaat uit vier onderdelen: algemeen, bron, doel en tegenrekening.</span><span class="sxs-lookup"><span data-stu-id="e3708-119">Each allocation rule has four components: general, source, destination, and offset.</span></span> <span data-ttu-id="e3708-120">Een extra onderdeel, de basisregels voor grootboektoewijzing, is vereist als Basis als toewijzingsmethode wordt gebruikt.</span><span class="sxs-lookup"><span data-stu-id="e3708-120">An additional component, ledger allocation bases rules, is required if Basis is used as the allocation method.</span></span> <span data-ttu-id="e3708-121">Al deze onderdelen bevatten essentiële informatie die nodig is om toewijzingen te verwerken.</span><span class="sxs-lookup"><span data-stu-id="e3708-121">Each component provides a critical piece of the information that is required in order to process allocations.</span></span>

-   <span data-ttu-id="e3708-122">**Algemeen** – In dit onderdeel geeft de gebruiker onder andere opties op voor de toewijzingsmethode, intercompany-regelinstellingen en of de regel actief is.</span><span class="sxs-lookup"><span data-stu-id="e3708-122">**General** – This component is where the user specifies options such as the allocation method, intercompany rule settings, and whether the rule is active.</span></span>
-   <span data-ttu-id="e3708-123">**Bron** - In dit onderdeel geeft de gebruiker de brongegevens voor de toewijzing op.</span><span class="sxs-lookup"><span data-stu-id="e3708-123">**Source** – This component is where the user specifies the source data for the allocation.</span></span> <span data-ttu-id="e3708-124">Toewijzing kan zijn gebaseerd op grootboeksaldi (**Gegevensbron** = **Grootboek**) of vaste bedragen (**Gegevensbron** = **Vaste waarde**).</span><span class="sxs-lookup"><span data-stu-id="e3708-124">Allocation can be based on ledger balances (**Data source** = **Ledger**) or fixed amounts (**Data source** = **Fixed value**).</span></span> <span data-ttu-id="e3708-125">Wanneer **Gegevensbron** is ingesteld op **Grootboek**, moeten bronfiltercriteria zijn gedefinieerd voor de grootboektoewijzingsregel (bijvoorbeeld voor de reclameonkosten).</span><span class="sxs-lookup"><span data-stu-id="e3708-125">When **Data source** is set to **Ledger**, source filter criteria must be defined for the ledger allocation rule (for example, for the advertising expenses).</span></span>
-   <span data-ttu-id="e3708-126">**Doel** – Via dit onderdeel bepaalt u hoe het resultaat van de toewijzingsberekening moet worden verdeeld en verantwoord.</span><span class="sxs-lookup"><span data-stu-id="e3708-126">**Destination** – This component defines how the result of the allocation calculation should be distributed and accounted for.</span></span> <span data-ttu-id="e3708-127">Er kan bijvoorbeeld één doelregel voor elke afdeling zijn.</span><span class="sxs-lookup"><span data-stu-id="e3708-127">For example, there can be one destination line for each department.</span></span>
-   <span data-ttu-id="e3708-128">**Tegenrekenen** - Deze component bepaalt hoe hoofdrekeningen en dimensies moeten worden bepaald voor de tegenrekeningsitems die de doelitems in overeenstemming brengen.</span><span class="sxs-lookup"><span data-stu-id="e3708-128">**Offset** – This component defines how main accounts and dimensions should be determined for the offset entries that balance the destination entries.</span></span> <span data-ttu-id="e3708-129">Door de gebruiker gedefinieerde opties worden meestal gebruikt in plaats van rekeningen en dimensies die zijn gebaseerd op de bron.</span><span class="sxs-lookup"><span data-stu-id="e3708-129">User-defined options are typically used instead of accounts and dimensions that are based on the source.</span></span> <span data-ttu-id="e3708-130">Wanneer **Gegevensbron** is ingesteld op **Vaste waarde**, kan **Bron** niet als optie worden gebruikt.</span><span class="sxs-lookup"><span data-stu-id="e3708-130">When **Data source** is set to **Fixed value**, **Source** can't be used as an option.</span></span>
-   <span data-ttu-id="e3708-131">**Basisregels voor grootboektoewijzing** - Deze regels gebruiken hun eigen bronfiltercriteria om te bepalen welke grootboeksaldi voor toewijzing (bijvoorbeeld de opbrengsten per afdeling) moeten worden gebruikt.</span><span class="sxs-lookup"><span data-stu-id="e3708-131">**Ledger allocation basis rules** – These rules use their own source filter criteria to determine which ledger balances should be used for allocation (for example, the revenue per department).</span></span> <span data-ttu-id="e3708-132">Elke toewijzingsbasisregel kan met meerdere toewijzingsregels worden gebruikt.</span><span class="sxs-lookup"><span data-stu-id="e3708-132">Each allocation basis rule can be used with multiple allocation rules.</span></span>
